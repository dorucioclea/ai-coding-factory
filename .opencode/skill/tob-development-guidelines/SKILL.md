# TOB Development Guidelines

A collection of Trail of Bits development guideline skills for secure software development.

## Sub-Skills

This skill contains multiple specialized sub-skills:

| Sub-Skill | Purpose |
|-----------|---------|
| [audit-prep-assistant](audit-prep-assistant/skill.md) | Prepare codebases for security audits |
| [code-maturity-assessor](code-maturity-assessor/skill.md) | Assess code maturity and production readiness |
| [guidelines-advisor](guidelines-advisor/skill.md) | Comprehensive development advisory based on TOB best practices |
| [secure-workflow-guide](secure-workflow-guide/skill.md) | Secure development workflow guidance |
| [token-integration-analyzer](token-integration-analyzer/skill.md) | Analyze token integrations for security issues |

## When to Use

Use these skills when:
- Preparing for a security audit
- Assessing code maturity before deployment
- Reviewing smart contract development practices
- Implementing secure development workflows
- Analyzing token integrations

## Usage

Invoke the specific sub-skill based on your needs:

```
# For audit preparation
Use audit-prep-assistant skill

# For code maturity assessment
Use code-maturity-assessor skill

# For comprehensive development guidance
Use guidelines-advisor skill

# For secure workflow implementation
Use secure-workflow-guide skill

# For token integration analysis
Use token-integration-analyzer skill
```

## Framework

Based on Trail of Bits' Building Secure Contracts guidelines and security best practices.
